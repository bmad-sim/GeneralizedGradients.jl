How the GG calculation works¶

This page summarizes the mathematics behind gg_fit. The field-expansion table is linear in the GG functions and their s-derivatives, which makes the fit a (weighted) linear least-squares problem.

The field expansion¶

For each field component c∈{Bx,By,Bs}c \in \{B_x, B_y, B_s\}c∈{Bx​,By​,Bs​},

Bc(x,y,z)=∑(n,m)CSc,b(n,m;x,y) b(n,m)(z)+∑(n,m)CSc,a(n,m;x,y) a(n,m)(z)+∑mCSc,bs(m;x,y) bs(m)(z),B_c(x,y,z) = \sum_{(n,m)} CS_{c,b}(n,m; x,y)\, b_{(n,m)}(z) + \sum_{(n,m)} CS_{c,a}(n,m; x,y)\, a_{(n,m)}(z) + \sum_{m} CS_{c,bs}(m; x,y)\, bs_{(m)}(z),Bc​(x,y,z)=(n,m)∑​CSc,b​(n,m;x,y)b(n,m)​(z)+(n,m)∑​CSc,a​(n,m;x,y)a(n,m)​(z)+m∑​CSc,bs​(m;x,y)bs(m)​(z),
(1)

where each coefficient is a polynomial in the transverse coordinates,

CSc,f(n,m;x,y)=∑(coeff⋅gref k xp yq),CS_{c,f}(n,m; x,y) = \sum (\text{coeff}\cdot g_{\text{ref}}^{\,k}\, x^p\, y^q),CSc,f​(n,m;x,y)=∑(coeff⋅grefk​xpyq),
(2)

and the derivative towers are b(n,m)=dmbn/dzmb_{(n,m)} = \mathrm{d}^m b_n/\mathrm{d}z^mb(n,m)​=dmbn​/dzm, a(n,m)=dman/dzma_{(n,m)} = \mathrm{d}^m a_n/\mathrm{d}z^ma(n,m)​=dman​/dzm, and bs(m)=dm+1a0/dzm+1bs_{(m)} = \mathrm{d}^{m+1} a_0/\mathrm{d}z^{m+1}bs(m)​=dm+1a0​/dzm+1.

Plane-by-plane least squares¶

The unknowns at a base plane z0z_0z0​ are the function values and their derivatives f(n,m)(z0)f_{(n,m)}(z_0)f(n,m)​(z0​) for m=0…mmax⁡m = 0 \ldots m_{\max}m=0…mmax​. The field on a neighbouring plane at offset dz=z−z0\mathrm{d}z = z - z_0dz=z−z0​ is obtained by Taylor-extrapolating each derivative,

f(n,m)(z0+dz)=∑j≥mdz j−m(j−m)! f(n,j)(z0),f_{(n,m)}(z_0 + \mathrm{d}z) = \sum_{j \ge m} \frac{\mathrm{d}z^{\,j-m}}{(j-m)!}\, f_{(n,j)}(z_0),f(n,m)​(z0​+dz)=j≥m∑​(j−m)!dzj−m​f(n,j)​(z0​),
(3)

which makes the model linear in the base-plane unknowns. Each base plane is then solved by weighted linear least squares over all field points lying within n_planes_add planes of it. Adding planes (m_max = 2·n_planes_add) lets the fit resolve higher derivatives and smooths the result; past some point, using more planes makes the polynomial approximation less accurate, so there is an optimum.

The merit function and weights¶

For a base plane, the merit function minimized is

Merit=∑weight (Btable−BGG)2,\text{Merit} = \sum \text{weight}\,(B_{\text{table}} - B_{\text{GG}})^2 ,Merit=∑weight(Btable​−BGG​)2,
(4)

with a per-point weight that factors as weight(x,y,dz)=wcore(x,y) wplane(dz)\text{weight}(x,y,\mathrm{d}z) = w_{\text{core}}(x,y)\, w_{\text{plane}}(\mathrm{d}z)weight(x,y,dz)=wcore​(x,y)wplane​(dz):

wcore(x,y)=core_weight rmax⁡2rmax⁡2+r2(core_weight−1),r2=x2+y2,w_{\text{core}}(x,y) = \texttt{core\_weight}\, \frac{r_{\max}^2}{r_{\max}^2 + r^2(\texttt{core\_weight}-1)}, \qquad r^2 = x^2 + y^2,wcore​(x,y)=core_weightrmax2​+r2(core_weight−1)rmax2​​,r2=x2+y2,
(5)
wplane(dz)=1+(outer_plane_weight−1) ∣dz∣dzmax⁡.w_{\text{plane}}(\mathrm{d}z) = 1 + (\texttt{outer\_plane\_weight}-1)\,\frac{|\mathrm{d}z|}{\mathrm{d}z_{\max}} .wplane​(dz)=1+(outer_plane_weight−1)dzmax​∣dz∣​.
(6)

core_weight = 1 (the default) weights all transverse points equally; a value

1 favours near-axis points, which is usually desirable since a beam spends most of its time near the core. outer_plane_weight = 1 weights all planes equally; a value below 1 (but non-negative) down-weights the outer planes. When n_planes_add = 0 (so dzmax⁡=0\mathrm{d}z_{\max} = 0dzmax​=0), wplanew_{\text{plane}}wplane​ is taken to be 1.

Conversion to Bmad’s convention¶

Bmad’s gen_grad_map uses azimuthal-harmonic gradients Cm,α(z)C_{m,\alpha}(z)Cm,α​(z), α∈{sin⁡,cos⁡}\alpha \in \{\sin, \cos\}α∈{sin,cos}, rather than this project’s midplane-derivative GGs. Equating the two expansions on the midplane gives exact recursions (solved in order of increasing mmm):

Cm,s[j]=1m![bm[j]−(m−1)!∑n≥1, m−2n≥1Wn(m,n) Cm−2n,s[j+2n]],C^{[j]}_{m,s} = \frac{1}{m!}\Big[ b^{[j]}_m - (m-1)! \sum_{n\ge 1,\, m-2n\ge 1} W_n(m,n)\, C^{[j+2n]}_{m-2n,s} \Big],Cm,s[j]​=m!1​[bm[j]​−(m−1)!n≥1,m−2n≥1∑​Wn​(m,n)Cm−2n,s[j+2n]​],
(7)
Cm,c[j]=1m![am[j]−(m−1)!∑n≥1, m−2n≥1Wc(m,n) Cm−2n,c[j+2n]−[m even] (m−1)! Us(m) bs[m+j−1]],C^{[j]}_{m,c} = \frac{1}{m!}\Big[ a^{[j]}_m - (m-1)! \sum_{n\ge 1,\, m-2n\ge 1} W_c(m,n)\, C^{[j+2n]}_{m-2n,c} - [m \text{ even}]\,(m-1)!\,U_s(m)\, b_s^{[m+j-1]} \Big],Cm,c[j]​=m!1​[am[j]​−(m−1)!n≥1,m−2n≥1∑​Wc​(m,n)Cm−2n,c[j+2n]​−[m even](m−1)!Us​(m)bs[m+j−1]​],
(8)
C0,c[j]=bs[j−1](j≥1),C^{[j]}_{0,c} = b_s^{[j-1]} \quad (j \ge 1),C0,c[j]​=bs[j−1]​(j≥1),
(9)

where x[j]≡djx/dsjx^{[j]} \equiv \mathrm{d}^j x/\mathrm{d}s^jx[j]≡djx/dsj is supplied directly by the fit. The full derivation, including the mixing weights WnW_nWn​, WcW_cWc​, and UsU_sUs​, is in the gg_to_bmad docstring in the API Reference.
